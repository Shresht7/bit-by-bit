Booleans

In computing, we often need to represent and manipulate logical values, which can be either true or false. A Boolean value can be represented using a single bit, where 0 typically represents false and 1 represents true.

Boolean Values

A Boolean is a value that can only be in one of two states. Depending on the context, we might describe those two states as true and false, yes and no, or on and off. The important part is not the label, but the fact that there are exactly two possible outcomes.

This makes Booleans a natural way to represent conditions and decisions. Is a button pressed? Is a user signed in? Did two values match? Questions like these do not need a large range of numbers. They only need a way to say "yes" or "no".

In hardware and software, we often encode those two states using bits. By convention, 0 usually means false and 1 usually means true. That convention is what lets the same underlying binary representation stand in for switches, flags, comparisons, and logical expressions.

Boolean Expressions

A Boolean expression is any expression that evaluates to either true or false. A single variable like A can be Boolean, but so can a compound expression like A AND B or NOT isEmpty.

Boolean expressions are the language of decisions in computing. They control branches in code, determine which gates activate in a circuit, and decide whether a particular condition has been satisfied.

Boolean Algebra

Boolean algebra is a system for combining and transforming Boolean values. Instead of working with quantities like ordinary arithmetic, it works with truth values and operators such as NOT, AND, and OR.

These operators are used everywhere in computing. The same ideas appear in programming languages, search filters, conditional statements, and digital circuits. Later, in the logic gates section, these operators will reappear as physical gate behaviors.

Core Operators

NOT flips a Boolean value. If A is true, then NOT A is false. If A is false, then NOT A is true.

AND is true only when every input is true. An expression like A AND B succeeds only when both conditions hold at the same time.

OR is true when at least one input is true. An expression like A OR B only becomes false when both inputs are false.

XOR, or exclusive OR, is true when the inputs differ. For two inputs, it answers the question "is exactly one of these true?"

Truth Tables

A truth table lists every possible input combination for a Boolean expression and shows the output produced by each one. It is one of the simplest ways to describe Boolean behavior precisely.

For one input, there are only two possible rows: the input can be 0 or 1. For two inputs, there are four possible rows: 00, 01, 10, and 11. As the number of inputs grows, the number of rows doubles each time.

Truth tables are especially useful because they let you compare expressions by behavior instead of by appearance. If two expressions produce the same output for every possible input, they are equivalent even if they look very different on the page.

Boolean Laws

Boolean algebra has a small set of laws that let us rewrite expressions without changing what they mean. These laws are useful for simplifying conditions in code, reducing logic in circuits, and proving that two expressions are equivalent.

Basic Laws

  • Identity: A AND 1 = A, A OR 0 = A
  • Null / Domination: A AND 0 = 0, A OR 1 = 1
  • Idempotent: A AND A = A, A OR A = A
  • Complement: A AND NOT A = 0, A OR NOT A = 1
  • Involution: NOT (NOT A) = A

Reordering Laws

  • Commutative: A AND B = B AND A, A OR B = B OR A
  • Associative: (A AND B) AND C = A AND (B AND C)
  • Associative: (A OR B) OR C = A OR (B OR C)

Distribution and Absorption

  • Distributive: A AND (B OR C) = (A AND B) OR (A AND C)
  • Distributive: A OR (B AND C) = (A OR B) AND (A OR C)
  • Absorption: A OR (A AND B) = A
  • Absorption: A AND (A OR B) = A

DeMorgan's Laws

  • NOT (A AND B) = (NOT A) OR (NOT B)
  • NOT (A OR B) = (NOT A) AND (NOT B)

Some of these laws are intuitive once you read them as statements about truth values, while others become much easier to trust after checking them with a truth table. DeMorgan's laws, in particular, are worth memorizing because they appear constantly when negating compound conditions.

Simplification Examples

The point of these laws is not just to memorize formulas. They let you transform a complicated expression step by step into one that is smaller, clearer, or easier to implement.

Example 1: Simplify A OR (A AND B)

A OR (A AND B) = A

This is the absorption law. Once A is already true, the extra term A AND B cannot make the expression any more true.

Example 2: Simplify NOT (A OR B)

NOT (A OR B) = (NOT A) AND (NOT B)

This is DeMorgan's law. Negating an OR expression changes the operator to AND and negates each input.

Example 3: Simplify (A AND 1) OR 0

(A AND 1) OR 0 = A OR 0 = A

First use the identity law for AND, then the identity law for OR. The result is just A.
